CapCom, Covoiturage

Une plateforme de covoiturage complete, construite en equipe de quatre en six semaines.

code source archive

Contexte & Objectifs

Pendant mon annee en BUT Informatique, ce projet collaboratif nous a demande de construire de zero une solution de covoiturage fonctionnelle : recherche de trajets, gestion des utilisateurs, reservations, profils conducteurs. Tout ca en six semaines, a quatre, avec Git comme terrain de coordination.

J'ai pris en charge l'integralite du CSS et de l'interface utilisateur : charte graphique, responsive design, composants reutilisables. Pendant que les autres membres geraient le backend PHP et la base MySQL, mon role etait de transformer les wireframes en pages utilisables.

  • BUT Informatique, projet universitaire
  • 6 semaines, equipe de 4 developpeurs
  • Role : lead CSS et integration UI
  • Enjeu : application web collaborative fonctionnelle
  • Stack : PHP, MySQL, HTML5, CSS3, JavaScript, Git

Apercu de l'Interface

Interface de recherche claire avec formulaire depart/arrivee et date.

Resultats de Recherche

Inscription

Bilan & Apprentissages

Prendre en charge tout le CSS d'un projet d'equipe, c'est plus complexe qu'il n'y parait. Il faut anticiper ce que les autres vont developper, creer des composants assez generiques pour etre reutilises, et maintenir la coherence visuelle pendant que le backend evolue en parallele.

Ce projet m'a aussi appris la valeur d'une organisation Git rigoureuse : branches par feature, revues de code, pas de push direct sur main. Ces reflexes acquis ici, je les ai gardes sur tous les projets suivants.

  • CSS modulaire et architecture de composants reutilisables
  • Responsive design sur mobile, tablette et desktop
  • Collaboration Git en equipe (branches, PR, resolution de conflits)
  • Communication entre developpeurs front et back pendant le developpement
